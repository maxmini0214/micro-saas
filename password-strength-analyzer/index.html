      Frequently Asked Questions

      What is password entropy and why does it matter?
      Password entropy measures randomness in bits: length × log2(charset_size). A password with 80+ bits of entropy would take billions of years to brute-force at 100 billion guesses/sec. For comparison: abc123 = ~31 bits (instant crack), X#9kP!2z@mQ = ~72 bits (centuries). More entropy = exponentially harder to crack.
      How long would it take to crack my password?
      This tool estimates crack time against two attack speeds: brute force (100 billion guesses/sec — GPU clusters like Hashcat on 8×RTX 4090) and smart attack (10 billion guesses/sec — dictionary + rule-based). A 20-char mixed password typically shows "centuries" for both. Real-world attackers use targeted rules first, so avoid dictionary words and patterns.
      Is my password sent to any server?
      No. Everything runs in your browser with zero network requests. Open DevTools → Network tab to verify: you'll see zero outgoing requests while typing. The common password list (70 entries) is embedded in the JavaScript. No analytics, cookies, or fingerprinting touch your password input.
      How do I check password strength in JavaScript?
      Calculate charset size: const cs = (/[a-z]/.test(pw)?26:0) + (/[A-Z]/.test(pw)?26:0) + (/[0-9]/.test(pw)?10:0) + (/[^a-zA-Z0-9]/.test(pw)?33:0). Then entropy: pw.length * Math.log2(cs). Check against common passwords with a Set, detect keyboard patterns (qwerty, 1234), repeated chars with /(.)\1{2,}/, and l33t speak by mapping a→4, e→3, etc.
      Is a passphrase better than a complex password?
      Generally yes. "correct-horse-battery-staple" (4 words from 2048-word list) = ~44 bits, but it's far more memorable than X#9k!2z. For maximum security, combine both approaches: use a passphrase and add symbols/numbers between words. In Python: import secrets; passphrase = '-'.join(secrets.choice(wordlist) for _ in range(5)).
      What makes a password weak even if it looks complex?
      Modern cracking tools check: common substitutions (p@ssw0rd = "password"), keyboard walks (qwerty, 1qaz2wsx), dictionary words + numbers (dragon123), repeated patterns (abcabc), and dates (19900101). This analyzer detects all of these. The best defense: true randomness via crypto.getRandomValues() or a password manager.
