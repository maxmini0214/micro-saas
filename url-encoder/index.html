❓ Frequently Asked Questions

What is the difference between encodeURI and encodeURIComponent?

encodeURI encodes a full URI but preserves special characters like :, /, ?, and #. encodeURIComponent encodes everything except letters, digits, and - _ . ~, making it ideal for encoding individual query parameter values.

How do I parse a URL into its components?

Use the URL Parser tab. Paste any URL and it instantly breaks it down into protocol, hostname, port, pathname, query parameters (each key-value pair), and hash fragment. Uses the browser's native URL API for 100% accuracy.

How do I build a query string from parameters?

Use the Query Builder tab. Add key-value pairs with the "+ Add Parameter" button. The tool auto-generates a properly encoded query string. Set a base URL to get the complete URL ready to copy.

Can I encode multiple URLs at once?

Yes! Enable Batch Mode in the Encode/Decode tab. Paste multiple lines and each line is encoded or decoded independently. Perfect for processing URL lists, CSV columns, or API parameter batches.

What is Deep Decode?

Deep Decode automatically detects and decodes multi-layer encoded URLs. If a URL was encoded twice (e.g., %2520 instead of %20), Deep Decode unwraps all layers until no more encoding is detected. Maximum 10 layers for safety.

Is this tool safe for sensitive data?

Yes. Everything runs in your browser. No data is sent to any server — verify in your browser's Network tab. See also: Base64 Encoder and Hash Generator.

When should I use URL encoding?

Use URL encoding when passing special characters in query parameters, form data, or API requests. Characters like &, =, ?, and spaces must be encoded to avoid breaking the URL structure.

What characters does encodeURIComponent NOT encode?

encodeURIComponent preserves A-Z, a-z, 0-9, and the characters - _ . ~. Everything else gets percent-encoded.
