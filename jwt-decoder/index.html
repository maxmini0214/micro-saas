Frequently Asked Questions

What is a JWT token? ▼
A JSON Web Token (JWT) is a compact, URL-safe token format used for securely transmitting information between parties. It consists of three base64url-encoded parts separated by dots: header (algorithm + type), payload (claims/data), and signature (verification).
Is this JWT decoder safe to use? ▼
Yes. This tool runs 100% in your browser using JavaScript. No data is sent to any server, ever. Your tokens never leave your machine. You can verify this by checking the Network tab in DevTools.
Can I create/encode a JWT token here? ▼
Yes! Switch to the "Build JWT" tab. Enter your claims (subject, issuer, custom key-value pairs), choose an HMAC algorithm (HS256/HS384/HS512), provide a secret key, and click Build. The signed token is generated entirely in your browser using the Web Crypto API.
What algorithms are supported? ▼
The encoder supports HMAC-based algorithms: HS256 (SHA-256), HS384 (SHA-384), and HS512 (SHA-512). The decoder can decode any JWT regardless of algorithm — RS256, ES256, EdDSA, etc.
How do I check if a JWT is expired? ▼
Paste your JWT in the Decode tab. If it contains an "exp" (expiration) claim, the tool automatically shows whether the token is valid (green) or expired (red) with the exact timestamp.
Can I verify the JWT signature? ▼
Yes. After pasting a JWT, a "Verify Signature" section appears. Enter the secret key and click Verify. The tool uses the Web Crypto API to check the HMAC signature against the token content.
What is the difference between decode and verify? ▼
Decoding extracts the header and payload (no secret needed — anyone can do it). Verifying checks that the signature matches using the secret key. Always verify in production; never trust decoded data alone.
Can I use this for RS256/ES256 tokens? ▼
The decoder works with any JWT. The encoder currently supports HMAC (HS256/384/512). RSA/EC signing requires private keys that are better handled in your own environment for security reasons.
